A lightweight, local-first hook made for offline-friendly React Native apps using SQLite via Supastash. Think of it as a trimmed-down version of useSupastashData ‚Äî no realtime updates, no global caching, just full control over filtering, pagination, and memory usage.

  • Clean result shape: flat data array + dataMap keyed by row id

‚ö†Ô∏è A Few Things to Know‚Äã

  • Your table must have a string id column ‚Äî it‚Äôs how we keep things deduplicated.

üß™ Example‚Äã

  useSupastashLiteQuery<Order>("orders", {

üß† Behind the Scenes‚Äã

  • Uses useReducer instead of useState for cleaner state transitions across pages

  • Filters are pre-built once on mount or when sqlFilter changes

  • Tracks each request so older responses don‚Äôt override fresh ones

  • Handles pagination smartly:

    • loadMore() fetches the next page
    • refresh() resets to page 0 and reloads

üõ°Ô∏è Built to Handle Edge Cases‚Äã

  • Prevents overlapping fetches (no double loads if you tap fast)
  • Handles rapid refresh/loadMore calls without messing up state
  • Ignores stale requests ‚Äî only the latest response updates the UI

üì¶ What You Get‚Äã

üßº Tips for Using It‚Äã
